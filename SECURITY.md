# 安全说明

本项目是自部署 Telegram Bot。部署前请先确认访问控制和凭据保存方式。

## 必须自建 Bot

- 请在 Telegram BotFather 创建自己的 Bot，并把 token 写入 `.env`。
- 不要把 Bot Token 提交到 GitHub、截图或发到群聊。
- 如果 Token 泄漏，请立即在 BotFather 轮换。

## 默认白名单

- 默认建议只允许自己的 Telegram 用户 ID。
- 如果开启公开访问，请确认部署机器、带宽、临时文件目录和隐私风险都能承受。
- 不建议把 Bot Token 和用户生成的媒体文件放进公开备份。

## 数据与临时文件

- Docker volume / `data` / `tmp` 目录可能包含缓存、下载文件或用户媒体。
- 迁移、备份、公开日志前请先检查是否包含隐私内容。
- 定期清理临时目录，避免磁盘被大文件占满。

## 上报问题

如果发现安全问题，请不要公开贴出 Token、用户 ID、私聊截图或服务器凭据。
